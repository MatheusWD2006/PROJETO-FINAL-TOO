Meu trabalho simula a questão das culturas agrículas e suas épocas do ano, levando em consideração o quanto uma determinada cultura se adapta a uma época do ano, ou até se ela pode ser plantada nas mais diversas situações do ano. O projeto permite que sejam criadas estações, culturas para essas estações ou para o ano todo, bem como fazer uma simulação do plantio e colheita delas. Ele é composto de 5 classes principais: Cultura, classe abstrata que traz objetos base de uma cultura; Estacao, que traz as estações do ano; CulturaEstacao, que herda de cultura e se associa com uma estação já existente; CulturaAnoTodo, herda de cultura e simula o plantio no ano todo e statusCultura que se associa com uma CulturaEstacao e simula o plantio dessa cultura. Também tem 2 arquivos de padrões de projeto, um para criação de objetos CulturaAnoTodo e CulturaEstacao, onde escondemos a lógica por trás deles e só instanciamos os objetos, e o outro é uma padrão Strategy para cria um aviso para o objeto aviso presente na classe Estacao. Esse aviso é gerado a partir do nome da estação. Esse projeto também tem 6 aruivos de teste, 1 para cada classe excluindo o strategy que está ligado ao teste das estações, o teste da abtrata serve para demonstrar o erro ao instanciar objetos para uma classe abstrata. Para execução é aconselhável executar os arquivos teste_estacao, o primeiro, já que exporta para teste_CulturaEstacao e testeFactory, depois teste_CulturaEstacao e testeFactory já que neles são criados objetos que vão ser exportados para o arquivo teste_StatusCulturaEstacao. Já os arquivos testeAbstrata e teste_CulturaAnoTodo aconselho a executar ou por primeiro ou por último, pra deixar limpo a execução dos testes que envolvem importação e exportação de objetos criados um com o outro. Mas também é possivel criar uma estação em teste_CulturaEstacao e em testeFactory, bem como criar uma CulruraEstacao em statusCulturaEstacao, dando mais uma possibilidade de execução
